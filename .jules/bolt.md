## 2025-05-18 - [Next.js App Router] Avoid `force-dynamic` on static-leaning index pages
**Learning:** The blog index page (`app/blog/page.tsx`) was completely opting out of static rendering with `export const dynamic = 'force-dynamic'` and `export const revalidate = 0`. This is a significant anti-pattern in Next.js App Router for relatively static lists, causing unnecessary server load (fetching all blog posts from the DB on every single hit) and high Time To First Byte (TTFB). Since the homepage and individual article pages already used Incremental Static Regeneration (ISR) with `revalidate = 60`, the blog list was an artificial bottleneck.
**Action:** When auditing performance, look for overly aggressive cache invalidation strategies (`force-dynamic`, `revalidate = 0`) on listing or index pages that don't need real-time freshness. Replace them with ISR (`revalidate = 60` or similar) to allow Next.js to serve fast cached responses while updating the cache seamlessly in the background.
